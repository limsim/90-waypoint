# 90 Waypoint Walk

A browser-based visualiser for the [90 Waypoint Walk](https://www.marcusjohnhenrybrown.com/the-90-waypoint-walk/) — a walking experiment created by Marcus John Henry Brown. The original walk was a 19 km stroll through Munich in August 2016, navigated by following a notebook of 90 left/right turns.

This app draws the walk as an orthogonal grid map. The turn sequence and segment lengths are both randomised on each generation, so every map is unique.

## How it works

The walker starts at the centre of the canvas facing North. Waypoint 1 is always oriented heading North — the path exits it straight upward. From the third waypoint onwards, a turn from the randomised sequence is applied at each step (Left = 90° counter-clockwise, Right = 90° clockwise), then the walker moves a random distance in the new heading.

**Wildcards** (scaled to roughly 1 per 9 waypoints) skip the scheduled turn — the walker continues straight ahead instead.

After all waypoints are placed, the path is centred on the canvas. If the layout cannot satisfy minimum separation constraints after 50 attempts, the canvas grows by 10% and generation retries.

## Getting started

**Prerequisites:** Node.js and Python 3.

```bash
npm install
npm run build
npm run serve
```

Open http://localhost:8000 in your browser.

During development, run the TypeScript compiler in watch mode in a separate terminal:

```bash
npm run dev
```

Changes to `src/index.ts` are compiled automatically; refresh the browser to see them.

## Controls

| Control | Description |
|---|---|
| **Generate Walk** | Draw a new map with a freshly randomised turn sequence and segment lengths. |
| **Clear** | Remove all waypoints and lines. |
| **Waypoints** | Number input (10–90, default 90). Sets the waypoint count for the next generation. |
| **Show Wildcards** | Toggle the orange rings that mark wildcard waypoints. On by default. |
| **Show Turns** | Toggle the L/R/W turn labels beside each waypoint. On by default. |
| **Print** | Open the browser print dialog; prints the canvas and legend on a single A4 page. |

Clicking a waypoint shows a tooltip with its number, turn direction, and cumulative distance from the start. Hovering highlights the waypoint and its adjacent path segments.

## Visual encoding

| Visual | Meaning |
|---|---|
| Black circle | Waypoint 1 (start) or last waypoint (end) |
| White circle | All other waypoints |
| Orange ring | Wildcard — walker went straight instead of turning |
| L / R / W label | Outbound turn direction at each waypoint (W = wildcard) |
| Number | Waypoint sequence number |
